TaskOps User Guides

This site aims to make onboarding with TaskOps as simple as possible. We will guide to through step by step, from defining your first Skill to setting up ongoing sprint scheduling and optimization.

Onboard Now

What is TaskOps?

TaskOps is a new way to handle task schduling for medium-sized enterprise businesses. We are a perfect fit for any company that generates a lot of tasks and needs to schedule them. TaskOps is based on the idea that task-scheduling is an optimization problem that can be solved using planning algorithms which enables happier more productive teams. TaskOps measures employee experience (total tasks they've completed ever), employee skills (you define which employees have which skills), priority (based on task priority) and availability that employees can set on the sprint calendar. TaskOps adjusts estimates by small factors to match experience and difficulty. We currently support Teamwork PM (Jira, Monday.com coming soon), which means your tasks stay where they are, in a single system but your employees get there own schedule address to check what they are working on that day.

Onboarding is a simple process.

You'll notice that the TaskOps main navigation bar is in order. Skills need to be created before Task Types are defined. Employees should be imported before importing tasks to a sprint. Using this guide, you should be able to get your company up and running in only a few minutes.
